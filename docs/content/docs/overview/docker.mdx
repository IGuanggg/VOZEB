# Docker 部署

如果你希望在自己的机器或服务器上运行项目，可以直接使用 Docker Compose。

## 使用发布镜像

项目根目录的 `docker-compose.yml` 已经指向 VOZEB 发布镜像：

```yaml
services:
  app:
    image: ghcr.io/csyqlz/vozeb:latest
```

```bash
git clone git@github.com:csyqlz/vozeb.git
cd vozeb
docker compose up -d
```

启动后访问：

```text
http://localhost:3000
```

## 本地构建镜像

如果需要基于当前源码构建镜像：

```bash
docker compose -f docker-compose.local.yml up -d --build
```

2 核 2G 服务器建议优先使用发布镜像；如必须本地构建，请使用支持 BuildKit 的 Docker 版本，构建缓存会复用依赖和 `.next/cache`。

## 文档站镜像

文档站位于 `docs/` 目录，按带服务端能力的 Next.js standalone 应用单独构建，不打进主应用镜像。

使用发布镜像：

```bash
cd docs
docker compose up -d
```

基于当前源码本地构建：

```bash
cd docs
docker compose -f docker-compose.local.yml up -d --build
```

## 数据说明

当前主应用镜像只启动 Next.js。画布、我的素材、生成记录和 AI API Key 默认保存在浏览器本地；公共提示词库会写入服务端 `.data/prompts.json`，如需长期保留管理员提示词，请挂载 `.data` 目录。
